Installing Microsoft Intune Portal on Debian 13 with a Local Installer Patch

A practical lab write-up documenting an unsupported workaround for patching the Microsoft Intune Portal Linux installer so Debian 13 is treated as Ubuntu 24.04 noble.

mormied Debian 13 Unsupported workaround
Important: This is an unsupported workaround. It may install successfully, but Microsoft support and Intune compliance behaviour may still depend on officially supported Linux distributions and desktop environments. Use this only after testing and only where you accept the risk.

Overview

Microsoft provides a Linux installer script for the Microsoft Intune Portal, which is designed to install the Intune app and its dependencies, including Microsoft Edge, on supported Linux distributions.

At the time of writing, Debian is not listed as a supported distribution. When running the official installer on Debian 13, the script exits early with an unsupported distribution error.

This write-up documents how to patch the installer locally so that it treats Debian 13 as Ubuntu 24.04, allowing the Microsoft Intune Portal installation to continue.

Tested Environment

  • Debian 13 “Trixie”
  • amd64 / x86_64 architecture
  • KDE desktop environment
  • Microsoft Intune Portal installer from the official Microsoft shell-intune-samples GitHub repository

What This Workaround Does and Does Not Do

This workaround only changes how the local installer script identifies the operating system during installation.

It does:

  • Allow the installer script to pass its distribution check
  • Point the Microsoft package source at the Ubuntu 24.04 noble repository
  • Avoid modifying /etc/os-release globally

It does not:

  • Make Debian an officially supported Intune platform
  • Guarantee successful device compliance evaluation
  • Guarantee future package compatibility
  • Change Microsoft support boundaries

The Problem

Running the Microsoft Intune Portal installer directly on Debian 13 fails because the script checks the distribution ID from /etc/os-release.

mkdir -p ~/intune-installer
cd ~/intune-installer

curl -L -o installer.sh "https://raw.githubusercontent.com/microsoft/shell-intune-samples/master/Linux/Intune%20Installer/installer.sh"
chmod +x installer.sh

./installer.sh --verbose

On Debian 13, the installer returns:

Microsoft Intune Portal Installer

Distro:       debian 13
Architecture: amd64
Channel:      prod
Log file:     /home/<user>/intune-installer.log

Error: debian is not a supported distribution. Supported: Ubuntu 22.04/24.04/26.04, RHEL/AlmaLinux 8/9/10

The issue is not necessarily that the package cannot run, but that the installer script refuses to continue because Debian is not in its supported distribution list.

Why Not Change /etc/os-release?

One option would be to temporarily spoof the whole operating system by editing /etc/os-release.

Do not do that.

Changing /etc/os-release system-wide can confuse package managers, system tools, scripts, and support tooling. A safer approach is to patch only a local copy of the installer script.

This keeps the workaround limited to the Intune installer and avoids changing the identity of the operating system globally.

Step 1: Download the Official Installer

Create a working folder and download the Microsoft installer script:

mkdir -p ~/intune-installer
cd ~/intune-installer

curl -L -o installer.sh "https://raw.githubusercontent.com/microsoft/shell-intune-samples/master/Linux/Intune%20Installer/installer.sh"
chmod +x installer.sh

Step 2: Create a Patched Copy

Create a copy of the installer script and patch the distribution detection section. The goal is to make the script behave as though the system is Ubuntu 24.04 “noble”, without changing Debian itself.

cd ~/intune-installer

cp installer.sh installer-debian-as-ubuntu.sh

python3 - <<'PY'
from pathlib import Path
import re

p = Path("installer-debian-as-ubuntu.sh")
s = p.read_text()

s2, n = re.subn(
    r'DISTRO="\$ID"\s+RELEASE="\$VERSION_ID"',
    'DISTRO="ubuntu"\nRELEASE="24.04"\nVERSION_CODENAME="noble"',
    s,
    count=1
)

if n != 1:
    raise SystemExit("Could not patch installer. Pattern not found.")

p.write_text(s2)
PY

chmod +x installer-debian-as-ubuntu.sh

Step 3: Confirm the Patch

Check that the script now forces the Ubuntu values:

grep -nE 'DISTRO=|RELEASE=|VERSION_CODENAME|CODENAME=' installer-debian-as-ubuntu.sh | head -30

Expected output should include:

DISTRO="ubuntu"
RELEASE="24.04"
VERSION_CODENAME="noble"

This part is important. Debian 13 uses the codename trixie, but the Microsoft Ubuntu 24.04 repository expects noble.

If the installer uses trixie with the Ubuntu 24.04 Microsoft repository, apt update will fail with a 404 error.

Step 4: Remove Any Broken Microsoft Repo Entry

If you previously ran a partially patched installer, it may have created an invalid Microsoft repository entry using trixie.

Remove the Intune-related Microsoft repo files before trying again:

sudo rm -f /etc/apt/sources.list.d/microsoft-prod.list
sudo rm -f /etc/apt/sources.list.d/microsoft-insiders-fast.list

Then check for any remaining bad entries:

grep -R "packages.microsoft.com/ubuntu/24.04/prod" /etc/apt/sources.list /etc/apt/sources.list.d/ 2>/dev/null

A broken entry would look similar to this:

https://packages.microsoft.com/ubuntu/24.04/prod trixie

The correct Ubuntu 24.04 component is:

noble

Step 5: Run the Patched Installer

Run the patched installer:

cd ~/intune-installer

./installer-debian-as-ubuntu.sh --verbose

The installer should now show something similar to:

Microsoft Intune Portal Installer

Distro:       ubuntu 24.04
Architecture: amd64
Channel:      prod
Log file:     /home/<user>/intune-installer.log

It should then continue installing prerequisites, adding the Microsoft repositories, updating package information, and installing the Microsoft Intune Portal package.

Step 6: Launch Microsoft Intune Portal

Once installed, launch the Intune Portal from the application menu. You can also try launching it from the terminal:

intune-portal

Sign in with the user’s work account and follow the enrolment process.

Troubleshooting

Check the Installer Log

If the install fails, check the log:

tail -n 100 ~/intune-installer.log

For a longer view:

less ~/intune-installer.log

Check the Microsoft Repository

If apt update fails, check the configured Microsoft repository:

grep -R "packages.microsoft.com" /etc/apt/sources.list /etc/apt/sources.list.d/ 2>/dev/null

For this workaround, the Intune Ubuntu repository should be using Ubuntu 24.04 and the noble component.

Check Package Availability

If the repo is correct but the install still fails, check whether the required packages are available:

apt-cache policy intune-portal microsoft-edge-stable

You can also check for common dependency issues:

apt-cache policy libwebkit2gtk-4.0-37 libjavascriptcoregtk-4.0-18

If dependencies are unavailable, do not add full Ubuntu base repositories to Debian. Mixing Debian and Ubuntu system repositories can break package management and leave the machine in a difficult-to-recover state.

Remove the Intune Portal

If you need to remove the installed package:

sudo apt remove intune-portal

To fully purge the package:

sudo apt purge intune-portal

You can also remove the Microsoft repository files added by the installer if they are no longer required.

Important Notes

This method only patches the installer script. It does not make Debian an officially supported Microsoft Intune Linux platform.

Administrators should be aware of the following:

  • Microsoft may not support this configuration.
  • Intune compliance results may not behave the same as on officially supported Linux distributions.
  • Future changes to the installer script may break this patch.
  • Future package dependency changes may prevent this from working.
  • This should be tested before being used on any production device.

The safest supported approach is still to use a supported Linux distribution.

Summary

The Microsoft Intune Portal installer blocks Debian because it is not listed as a supported distribution. Rather than modifying /etc/os-release, a safer workaround is to patch a local copy of the installer so that it treats Debian 13 as Ubuntu 24.04 “noble”.

The key part of the workaround is forcing:

DISTRO="ubuntu"
RELEASE="24.04"
VERSION_CODENAME="noble"

This allows the installer to add the correct Microsoft Ubuntu 24.04 repository and continue with the Intune Portal installation.

This is useful for lab environments, testing, or edge cases where Debian is required, but it should be clearly documented as unsupported.
